Situation

Our agency sent clients design mockups in a PDF format to review. This lead to the following problems:

  • Unprofessional. The PDF stretched to whatever screen you had.
  • Confusion lead to questions that fell on CX.
  • No archive or record of mockups.
  • Not scalable. Digging through emails never is.

Process

  1. Research:
    • Found a 3rd party service that solved problem for a while, then it went under.
    • I used their idea, took initiative to create one in-house.
  2. Built Prototype: HTML holder for 4 mockups, FTP back-end for designers to use.
  3. Beta test: Sent early version to a few clients and CX members for feedback.

    4. Sought out advice and critique from design team.

    Early version used one URL, but had no nav bar. Internal feedback said it wasn't easy enough to navigate around.

The first page

Solution

A bundle of static mockups with an interface and talking-points to set the stage.

This can be used to send to the client or as a tools while presenting the design.

Why this solution works

  • It puts each round of design into one URL
  • It puts the mockup in a realistic setting
  • It builds on conventions in the industry

What would I change?

  • Improve UX for designers, from FTP
  • Make menu dynamic if you want to swap pages depending on project
  • Include a send as text option so you can see it on your phone

Problem: Hard for client to review their mockup again

4 links usually means 4 new browser tabs. This can be hard difficult for the non-computer savvy user. Itâ€™s not as professional of a system as we can make it.

Solution: New interface

A friendly way for clients to navigate through mockups. Every project will have 1 main link, as a homebase.

Problem: Giving design feedback is hard

Clients are put into a place where they might not know how to express themselves. This might be their first situation having to discuss design. The better we understand their vision, the the better we can design it.

Solution: Tips for effective feedback

Add short suggestions for giving feedback to the intro page or to the client directly.

Metrics for success

  • Feedback: We received almost immediate positive feedback from the 2 stakeholders: CX and clients.
  • Saved time for CX
    • No more troubleshooting PDFs
    • Reinforced repeated talking points
  • Legacy: Used on 250+ designs & counting
